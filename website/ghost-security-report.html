🚨 Disaster Recovery
What If Everything Goes Offline?
Ghost Protocol is fully on-chain. If the website, bot, and all frontends disappear tomorrow — your assets are safe. Here's what you need to know.
The Short Version
💰
Your assets never leave Solana
Everything you deposit sits in a vault controlled by the Solana blockchain — not our servers, not our wallets, not anyone's wallet. As long as you have your Solana wallet, you can always recover your assets. No password resets, no support tickets, no dependence on us.
🌐
The website is just a window
Our website is a convenient interface, but it's not where your assets live. If ghost-protocol.rip goes down, your assets don't go with it. Anyone can build a new frontend that talks to the same on-chain program.
🤖
The executor bot is replaceable
Our bot monitors heartbeats and triggers execution automatically. If it goes offline, your ghost simply waits. Anyone can run their own executor bot — or trigger execution manually. The 5% bounty incentivizes this.
👥
Your guardians can help
If you set up recovery wallets (guardians), they can withdraw your vault assets at any time without needing the website. They just need their wallet and access to the Solana network.
Bottom line: Ghost Protocol has no single point of failure. No server to hack, no database to corrupt, no company to shut down. Your assets are on Solana, controlled by math, recoverable with your wallet.
How to Recover — By Scenario
Scenario What Happens Assets Safe?
Website goes down Use any Solana wallet + CLI tool to interact with the program directly. Your vault, ghost, and beneficiaries are all on-chain and unchanged. YES
Executor bot goes offline Your ghost stays in its current state (alive/awakened). No assets move. Anyone can run an executor or trigger execution manually when they come back online. YES
Ghost Protocol team disappears The on-chain program continues to run indefinitely. All functions remain callable by anyone. The program is deployed and immutable unless the upgrade authority acts. YES
You lose access to your wallet If you set up guardian wallets, they can withdraw your vault assets. Without guardians, assets remain locked in the vault — same as losing any crypto wallet. DEPENDS
Solana network goes down No transactions can be processed on any Solana dApp. Once the network resumes, your ghost, vault, and all on-chain state are exactly as you left them. Nothing is lost. YES
🔧 Technical Recovery Guide — For Developers
If you need to interact with the program without any frontend, here's the technical reference.
Prerequisites
1. Install the Solana CLI
2. Have access to your wallet keypair file (export private key from Phantom)
3. Set your RPC: solana config set --url https://api.mainnet-beta.solana.com
1 — Find Your Vault Addresses
const { PublicKey } = require("@solana/web3.js");
const PROGRAM = new PublicKey("3Es13GXc4qwttE6uSgAAfi1zvBD3qzLkZpY21KfT3sZ3");
const WALLET  = new PublicKey("YOUR_WALLET_ADDRESS");

const [ghostPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("ghost"), WALLET.toBytes()], PROGRAM
);
const [vaultPda] = PublicKey.findProgramAddressSync(
  [Buffer.from("vault"), WALLET.toBytes()], PROGRAM
);
console.log("Ghost:", ghostPda.toBase58());
console.log("Vault:", vaultPda.toBase58());
2 — Check Vault Balances
